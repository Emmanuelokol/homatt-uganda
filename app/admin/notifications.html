help_outline Setup Checklist
Follow these steps if push notifications are not working.
1
Create a OneSignal app
Go to app.onesignal.com → New App/Website → choose Mobile App → follow the Android FCM setup. Copy your App ID and REST API Key from Settings → Keys & IDs.
2
Add secrets to Supabase
Supabase Dashboard → your project → Edge Functions → Manage secrets.
Add these two secrets:
ONESIGNAL_APP_ID = your OneSignal App ID
ONESIGNAL_REST_API_KEY = your REST API Key
3
Add the same App ID to GitHub Secrets
GitHub repo → Settings → Secrets → Actions.
Add secret: ONESIGNAL_APP_ID = your OneSignal App ID.
This lets CI inject the App ID into the mobile app's config.js at build time.
4
Deploy the Edge Function
Push any change to supabase/functions/ on the main branch. The CI workflow will deploy send-notification automatically. Or run manually: supabase functions deploy send-notification --project-ref kgkdiykzmqjougwzzewi
5
Open the app and allow notifications
Launch the Homatt Android app and accept the push notification permission when it appears. Then sign in — the app registers your device token and saves it to your profile. Once saved, use the "Send Test Notification" button above to verify end-to-end delivery.
